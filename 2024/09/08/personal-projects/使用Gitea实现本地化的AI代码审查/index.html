3. 部署项目

    正好还有一台无界s也处于吃灰状态，就部署到这上面吧。再加上我在建项目的时候勾了GraalVM Native Support选项，我决定尝试一下构建native-iamge。

    因为我不想在服务器上安装gradle、kotlin环境，就想着本地打个jar包后再将jar包转成native-image，然后我问了半天GPT都无法解决找不到MainClass的问题，最后去翻Spring的文档找到了一串脚本命令，试了一下后果真能行。

Spring官方文档还列出了此方式的优点，见Advanced Native Images Topics :: Spring Boot

或者直接看中文文档：GraalVM 原生镜像（Image）支持 (springdoc.cn)

    把jar包上传到服务器然后运行脚本就能一键构建并启动了

4. 效果

  

    当我提交推送到本地仓库时，会触发预先定义好的Web 钩子，即请求上面部署好的AI代码审查服务的接口，然后执行AI代码审查，将审查结果保存到数据库后，推送微信模板消息

     

    点击查看详情，就能跳转到AI代码审查结果页面

5. 总结

    通过上面的努力，基本实现了一个可用的AI代码审查服务（写提示词真是个技术活）。个人觉得后续可优化的点如下：

  1. 前端页面优化

  2. 提示词（Prompt）持续优化

  3. 可以通过内网穿透将代码仓库和AI代码审查服务暴露到公网，以便随时随地提交代码及接收和查看代码审查结果
